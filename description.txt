Общие задачи:
1. сайт адаптивный. Breakpoints - '640px', '768px', '1024px', '1280px'.
2. Данные подгружаются с помощью AJAX из файла JSON загружаются из Firebase. Если при первой попытке соединения с БД не удалось загрузить данные, то через 3 сек. происходит повторное соединение с БД и так до тех пор, пока с БД не удастся соеденится
3. Фильтры в виде выпадающих списков.
4. Фильтрация услуг происходит по всем имеющимся свойствам и их значениям в списке услуг.
5. Описание услуги в списке не должно превышать 100 символов.
6. Изображение - ссылка на страницу услуги.
7. Картинки перенесены в Storage Firebase, файл JSON перенесен в RealTime DataBase. После перехода на главную страницу происходит загрузка из Firebase
8. Для более привлекательного отображения на мобильных устройствах, фильтр вместо <select> заменен на обычный список, при этом основные возможности <select> сохранены:
	- при клике на название фильтра либо на кнопку в виде галочки, открывается контекстное меню
	- при клике в любом месте страницы, в т.ч. и на другие поля фильтра, фильтр, который был открыт - автоматически закрывается
	- при повторном клике на название фильтра либо на кнопку в виде галочки, контекстное меню закрывается
	- при выборе пункта контекстного меню, данный пункт отображается вместо названия фильтра, контекстное меню автоматически закрывается
9. создан favicon
10. Создан Readme.md файл с инструкцией о загрузке, развертывании, компиляции (и т.д.) приложения.

Страница услуги
11. На страницу услуги можно попасть либо со страницы списка услуг, кликнув на изображение в списке услуг, либо по прямой ссылке вида: "http://you_domain/service/service_alias"
12. Описание услуги полное.
13. Цена услуги имеет формат целого числа разделенного пробелом.
14. Кнопка "Купить" добавляет в "корзину" приобретенную услугу (здесь обычный счетчик).
